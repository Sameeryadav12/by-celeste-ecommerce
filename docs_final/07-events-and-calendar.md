# Events & Calendar

## Purpose

This document will explain how events will be listed, promoted, and managed (for example, markets, pop-ups, or launches).

## What the system stores now (Step 10)

- Events are stored in the backend with practical details for customers:
  - title, slug, short description, and full description
  - location name + address (optional lines)
  - start and end date/time
  - optional image
  - `isPublished` (public vs hidden) and `isFeatured` (highlight)

## What customers can see now

- The **Events** page shows:
  - a **featured upcoming** event (when available)
  - a grid/list of **upcoming** events
  - search by title (simple filter)

- Each event card links to an **event detail** page where customers can read the full description and see the full location and date/time.

## What will later be added

- RSVP or ticketing (if needed)
- A fuller calendar view
- The admin dashboard now supports staff event creation, editing, and publish/unpublish controls.
